Git搭建及常用命令

  • Git
  • 后端相关
  • 常用命令
  • 教程

Git对于独立开发者来说，我认为更多的是为了代码安全，比如代码备份，比如代码回溯，作用地位并不是那么的明显；而对于团队来说，就显得至关重要了。协同开发是一个学生向职业程序员转变的一个重要标志，而Git又是其中的佼佼者。本文将介绍Git服务器的搭建、Git的常用命令以及两款可视化工具。

本文只做简单介绍，只讲述了最基础最常用的，需要进一步说明的可参照：廖雪峰git教程以及Git文档。

Git服务器搭建

安装Git

如果你的系统是ubuntu，可以直接安装

$ sudo apt-get install git

或者也可以自行下载安装：Git Download

配置Git

  1. 添加git用户

    $ sudo adduser git

  2. 初始化Git仓库

    选择一个目录作为Git仓库，通常是在/srv目录下，如要初始一个名为test的仓库，可在/srv目录下输入：

    $ sudo git init --bare test.git

  3. 更改owner

    一般情况下，初始化一个Git仓库后，没有权限Pull或者Push，问题就在于没有更改owner：

    $ sudo chown -R git:git test.git

  4. 禁用shell登陆

    通常情况下，出于安全考虑，git用户hi不允许登陆shell的，这个可以通过修改/etc/passwd完成。vi打开/etc/passwd，将

    git:x:1001:1001:,,,:/home/git:/bin/bash

    改为

    git:x:1001:1001:,,,:/home/git:/usr/bin/it-shell

  5. 公钥证书

    将所有项目相关人员公钥导入/home/git/.ssh/authorized_keys文件中，没有的话就创建，一行对应一个公钥。

    公钥通过ssh-keygen生成，位置是当前用户家目录下.ssh文件夹中，id_rsa.pub就是我们所需要的公钥文件。

Git常用命令

初始化

$ git init

克隆

$ git clone git@192.168.1.117:/srv/test.git test

命令中最后的test为克隆目标文件夹，缺省默认对应Git仓库名。

Pull & Push

Pull远端develop分支上代码

$ git pull origin develop

将本地修改代码提交至远端develop分支上

$ git push origin develop

通常提交时还可能会用到强制提交，添加－f参数即可。

分支

由当前分支切换到develop分支：

$ git checkout develop

本地新建并切换至develop分支：

$ git checkout -b develop

本地删除develop分支：

$ git branch -d develop

提交修改

将修改添加至git中：

$ git add modify_file

修改过多时，也可直接用.替换modify_file

为提交的修改做注释

$ git commit -m "It's my first commit."

版本回退

当版本出现问题，不可修复或修复成本过高时，可以考虑版本回退到上一个有效版本

$ git reset --hard HEAD^ //回退到上一个版本
$ git reset --head 版本号 //回退到版本号对应的版本

Git可视化工具

tig

tig是一款git资源浏览器，可以很方便的在终端下查看所有的提交纪录。

ubuntu可以直接通过apt-get安装：

$ sudo apt-get install tig

也可以通过编译安装：下载 tig

gitk

相对于tig终端下的资源浏览，gitk就是一款独立的可视化工具了。
