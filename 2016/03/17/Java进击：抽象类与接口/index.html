Java进击：抽象类与接口

  • Java进击
  • 常用概念
  • 语法

抽象类与接口时java中对抽象概念进行定义的两种机制，但对于新手来说，这两个概念存在着很大的相似，甚至可以互换，有些难以理解。本文将对抽象类与接口做出解释与对比，以作备忘。

抽象类

什么是抽象类

我们都知道，在面向对象的概念中，所有对象都是通过类描述的。但是反过来，并不是所有的类都可以用来描述对象的。如果一个类中没有包含足够的信息来描述一个具体对象，这样的类就是抽象类。

从语法上来讲，使用了关键字abstract修饰的类就称为抽象类。通常情况下，如果一个类中包含了一个或多个抽象方法，那么这个类就必须指定成抽象类。如下，我们定义了一个Animal的抽象类，有两个抽象方法，分别来表征动物的奔跑以及叫这两个属性：

为什么要用抽象类

抽象类除了不能实例化对象之外，类的其他功能依然存在，成员变量、成员方法以及构造方法的访问方式均和普通类一致。

抽象类定义了一组抽象方法，但这组抽象方法的具体实现只能由派生类实现，这种设计方式体现了数据抽象的思想，是实现多态的一种机制。同时，抽象类不能实例化对象，因此抽象类必须被继承才能够被使用。正是由于这个原因，通常在类设计阶段就决定了是否设计及采用抽象类。

抽象类使用场景：

  1. 如果拥有一些方法，并且希望这些方法中有一些可以默认实现；
  2. 如果一些方法的基本功能不断改变，那么就需要使用抽象类，这样可以避免在实现该方法的多处地方频繁变动。

抽象类注意问题

在使用抽象类时，我们通常需要注意几个问题：

  1. 只要包含抽象方法的类，就必须定义成抽象类，不管是否包含其他方法；
  2. 抽象类不能实例化，只能由其子类实例化；
  3. 抽象方法必须由子类进行重写；
  4. 抽象类可以包含具体方法，同时，也可以不包含抽象方法；
  5. 子类中的抽象方法不能与父类抽象方法相同；
  6. abstract不能与final同时用来修饰同一个类；
  7. abstract不能与private、static、final或native并列修饰同一个方法。

接口

什么是接口

接口实际上就是一系列方法的声明，或者说是一些方法特征的集合。接口只有方法的特征而没有方法的实现，因此这些方法可以在不同的地方被不同的类实现。

那么接口到底是什么？接口的最大意义实际上就是告诉你这个类里有啥，能干啥。举个例子来说，我们看到肯德基就知道里面有炸鸡，实际上接口也是这样的，接口就是告诉你里面有什么行为，至于怎么实现这些行为就需要你自己来搞了。比如说，我们希望完成一个字符串的比较，我们可以在类中直接定义一个方法完成字符串的比较，但在另一个类中我又想完成一个整数的比较呢？很多情况下，人们是不关注具体实现过程的，他们只关注做了些啥事儿，而接口就可以达到这样的目的。

还有一种理解，可以把接口认为是一个统一标准，或者就是个协议（在Object-C中已经把接口称作协议了）。这样的目的，或者好处就是上面所说的：能够让大家知道这个是做什么，而又不需要知道具体怎么做。

什么时候使用

如果希望实现多重继承，那么就必须采用接口了。因为Java不支持多继承，子类不能够继承多个类，但可以实现多个接口。

接口使用注意问题

使用接口过程中需要注意如下几个问题：

  1. 接口中所有方法访问权限均被自动声明为public，否则，将会编译出错；
  2. 接口中不存在实现的方法；
  3. 接口中可以定义成员变量，但所有成员变量会自动被public staic final修饰，也就是不可变变量；
  4. 实现接口的非抽象类必须实现所有方法，抽象类可以不用实现；
  5. 不能使用new实例化一个接口；
  6. 多接口实现时注意避免方法名的重复。

区别

  1. 从语法层次来说，抽象类可以拥有任意范围的成员数据，同时也可以拥有自己的非抽象方法；但在接口中，仅能够有静态、不能修改的成员数据，并且其所有方法都必须是抽象的。从某种程度上来说，我们可以认为接口是抽象类的特殊化。同时，从继承角度来说，子类只能继承一个抽象类，但可以实现多个接口；
  2. 抽象层次不同，抽象类是对类的抽象，而接口是对行为的抽象，这是一个整体与局部的关系；
  3. 抽象类的子类一般都是具有相似特点的，比如说，猫、狗同属于动物；而对于接口来说，它的实现子类不存在任何关系，最多我们可以认为这些子类有着部分相类似的功能；
  4. 抽象类可以有构造器，而接口不能有构造器；
  5. 相比较来说，抽象类的速度要比接口块，最主要原因是接口需要时间寻找类中实现的方法。
