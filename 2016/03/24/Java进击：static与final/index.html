Java进击：static与final

  • Java进击
  • 常用概念
  • 语法

本文对Java中static与final关键字的使用做出了总结，以作备忘。

final关键字

final表示“不可改变”或者“终态”的意思，可以用来修饰非抽象类、非抽象类成员方法和变量。

final类

final类不能被继承，这就意味着final类的成员方法没有机会被覆盖；final类中的方法默认时final的。
通常，在进行类设计时，如果不需要子类，类的实现细节不需要改变，并且不会再被扩展，那么该类就可以被设计成final类。

final方法

final修饰的方法也被称为最终方法，即不允许被子类重新定义，也不允许被覆盖。也就是说，当一个类中存在着不允许被其子类覆盖的某个方法，那么这个方法就可以声明为final方法。但需要注意的是，不能被覆盖并不意味着不能被重载。
使用final方法通常有两个好处：第一，防止任何继承类修改现有方法的实现过程；第二，编译器在处理final方法时，会将其转入内嵌机制，从而大大提高执行效率。

final变量

final变量表示常量，也就是说final修饰的成员变量，一旦赋值，将无法改变。
需要注意的是，final变量的定义是允许先定义而不给初值的，这种变量成为final空白。但必须保证其在使用前初始化。这种机制的存在为final的使用提供了很大的灵活性：可以保证类中的final变量根据对象的不同而有所不同，却又能够保证其不变的特性。
final变量通常也会加static关键字，这样实际上就是充分利用二者的特性来减少内存空间的使用。

小结：我们可以看出final修饰的属性本质的特征就是恒定不变。

static关键字

static表示“全局”或“静态”的意思，可以用来修饰成员变量和成员方法，也可以形成静态代码块。这里需要注意的是，在Java中是没有全局变量的概念的。

static变量（静态变量）

Java中变量按照是否被static修饰可以分为静态变量和实例变量，静态变量也叫作类变量。
静态变量在内存中只有一个拷贝，这是因为JVM在加载类的过程中，只为静态变量分配一次内存，也就是说无论是该类创建了多少个对象，甚至是不创建对象，该数据区域均存在且唯一，这种机制可以大大节省内存空间。静态变量可以用类名直接访问，也可以通过对象访问，事实上，也仅仅是访问方式的不同，而访问对象、访问结果均相同。
静态变量并非是不可改变的，相反的是，静态变量一经改变，其值将会在所有对象中随之改变。

static方法（静态方法）

与静态变量相同，静态方法在内存中也只有一个拷贝，不与该类的任何对象相关联，可以通过类名直接调用，也可以通过实例调用。
不同的是，静态方法不能直接访问所属类的实例变量和实例方法，也不能采用this和super关键字，它只能访问所属类的静态成员变量和静态方法；由于独立于任何实例这一特点，静态方法必须被实现，而不能够是抽象的abstract。
事实上，main函数可以有效地帮助我们理解静态方法。类中存在的main函数均是被static修饰的，也就是静态方法，JVM在运行main函数的时候，是直接调用的，而不是创建实例。

static代码块（静态代码块）

静态代码块是独立于类成员的static语句块，可以有多个，位置也可以任意，但需要说明的是，静态代码块不能存在于任何的方法体内。
JVM在加载类的过程中会执行静态代码块；对于多个静态代码块，JVM会按照它们在类中出现的先后顺序依次执行。需要强调的是，所有的静态代码块只会被执行一次。

小结：我们可以看出，无论是static修饰方法，变量还是代码块，其共同特点就是被static修饰的属性在内存中均只有一个拷贝，不与任何实例相关，这就是static关键字的最本质作用。

final与static区别

  1. 首先我们需要明白的是：static与final事实上没有啥直接关系。

  2. final可以修饰类，而static只能修饰成员方法与变量。

  3. 看到最多的两者联系可能就是在于final与static通常联用，二者联用产生的效果可以简单地理解为“全局”，目的就是为了减少内存空间的使用。
